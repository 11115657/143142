# Realism Vertical Slice Upgrade v0.3

本版本不是商业级十几 GB 游戏资源包，而是 GitHub Pages 可运行范围内的“真实游戏纵切原型”。商业游戏体积主要来自高精度模型、4K/8K 贴图、动作捕捉动画、语音、音乐、剧情视频和多地图资源。当前工程刻意不内置这些资产，避免侵权和无意义体积膨胀。

## 已升级的真实游戏系统

- 更大的程序化训练基地：围墙、塔楼、指挥楼、仓库、集装箱区、车辆区、靶场、补给点、撤离点。
- 三把武器：步枪、冲锋枪、手枪，包含射速、伤害、射程、后坐力、瞄准散布、穿甲、装填时间。
- 四类敌人：步枪兵、突入兵、重甲兵、精确射手，包含护甲、反应时间、命中率、爆发射击、偏好距离。
- AI 感知：距离、遮挡射线、警戒值、搜索、追击、交火、低血量找掩体。
- 玩家生存：生命、护甲、体力、奔跑消耗、跳跃消耗、医疗包。
- 战术道具：破片手雷，包含投掷、重力、反弹、延迟爆炸、范围伤害。
- 任务链：清理外围、访问情报终端、撤离。
- 补给交互：弹药箱、护甲箱、医疗箱。
- HUD：生命、护甲、体力、医疗包、手雷、武器、弹药、任务、交互提示、命中反馈、调试面板。
- Blender 插槽：当前没有 GLB 时用程序化资产运行，有 GLB 时可逐步替换。

## 资产体积为什么仍然不会很大

代码和程序化几何体天然很小。真正让游戏达到十几 GB 的内容应来自你后续制作并导入的 Blender 资产：

- 多个高质量角色 GLB：每个 10-80MB
- 武器模型和动画：每把 5-30MB
- 场景 GLB：每张地图 100MB-2GB，Web 端需要进一步拆分和压缩
- PBR 贴图：每套 2K/4K 贴图 5-80MB
- 动画库：移动、射击、换弹、受击、死亡，每套数 MB 到数十 MB
- 音效和音乐：数十 MB 到数 GB

GitHub Pages 项目不建议塞入十几 GB 资源。正确方式是：GitHub Pages 放试玩版和轻量资源，大资源走 Release、CDN、对象存储或客户端安装包。

## 当前版本的定位

v0.3 是 GitHub Pages 能承受的高完成度浏览器原型，而不是完整商业客户端。它的价值是：

1. 玩法闭环已存在。
2. 文件结构可继续扩展。
3. 所有关键系统都有入口。
4. 你可以用 Blender 逐个替换占位资产。
5. 后期可接多人服务器或 PC 客户端版本。
